SkinSense API Documentation

Referensi lengkap REST API untuk sistem deteksi penyakit kulit berbasis AI — mencakup autentikasi, deteksi, rekomendasi AI, dan forum komunitas.

🌐 REST / JSON
🔥 Backend: Node.js + Express
🗄️ Database: Firestore
🖼️ Storage: Cloudinary
🤖 AI: Gemini 3.5
📖 Introduction

API ini adalah backend untuk aplikasi SkinSense — sebuah platform deteksi penyakit kulit berbasis AI yang dilengkapi fitur forum komunitas. Seluruh endpoint menggunakan format JSON dan mengikuti konvensi REST standard.

Base URL (Dev) http://localhost:3000
Base URL (Prod) https://<your-domain>
🔐 Authentication

API menggunakan dua jenis middleware autentikasi: requireAuth dan optionalAuth. Keduanya mengharapkan header Authorization: Bearer <token>.

🔒 requireAuth
Wajib login. Hanya menerima Firebase ID Token yang valid. Digunakan pada endpoint yang membutuhkan identitas penuh (buat post, buat komentar, dll).
🔓 optionalAuth
Mendukung dua jenis token: Firebase ID Token (user login) atau Guest Token yang diawali kata guest (contoh: guest_abcd1234). Digunakan pada deteksi dan baca forum.
💡
Cara mendapatkan Firebase ID Token
Di Flutter/Android/iOS, gunakan FirebaseAuth.instance.currentUser?.getIdToken(). Token berlaku selama 1 jam dan harus di-refresh secara periodik. Untuk guest user, gunakan token string unik yang diawali guest (misalnya UUID yang di-prefix).
ℹ️
Format Header Authorization
Semua request yang membutuhkan autentikasi harus menyertakan header:
Authorization: Bearer <firebase_id_token_atau_guest_token>
📦 Response Format

Semua response dari API menggunakan format JSON yang konsisten seperti berikut:

JSON · Standard Envelope 
{
  "code": 200,            // HTTP status code
  "status": "success",    // "success" jika code < 400, "failed" jika >= 400
  "message": "...",      // Pesan deskriptif dalam Bahasa Indonesia
  "data": null | {} | [] // Payload data (null jika tidak ada data)
}
⚠️ Error Codes
Status CodeStatus TextPenyebab Umum

403ForbiddenUser mencoba edit/hapus resource milik user lain

}

🔬 Detection Service

Endpoint untuk deteksi penyakit kulit dan riwayat deteksi. Semua endpoint detection menggunakan optionalAuth — mendukung user login maupun guest.

ℹ️
Alur Upload Gambar
Frontend harus upload gambar ke Cloudinary terlebih dahulu, kemudian mengirimkan image_URL dan cloudinary_public_id hasil upload ke endpoint deteksi ini. Server tidak menerima file binary secara langsung.
POST /detection Buat deteksi baru, jalankan AI model optionalAuth ▼
Request Body (application/json)
FieldTipeValidasiKeterangan
image_URL string required URL gambar hasil upload ke Cloudinary (harus valid URI)
cloudinary_public_id string required Public ID dari Cloudinary untuk keperluan hapus gambar
Request Example
JSON
{
  "image_URL": "https://res.cloudinary.com/coffins/image/upload/v1778927926/main-sample.png",
  "cloudinary_public_id": "i1829a"
}

JSON
{
  "code": 201,
  "status": "success",
  "message": "Deteksi berhasil",
  "data": {
    "id": "auto_generated_doc_id_A",
    "label": "Acne",
    "confidence": 91.5,
    "image_URL": "https://res.cloudinary.com/coffins/image/upload/v1778927926/main-sample.png",

POST /detection/recommendation Dapatkan rekomendasi AI dari hasil deteksi optionalAuth ▼

Mengirimkan satu atau lebih hasil deteksi ke Gemini AI untuk mendapatkan rekomendasi penanganan, deskripsi penyakit, saran obat bebas, dan peringatan medis. Bisa digunakan setelah deteksi berhasil.

FieldTipeValidasiKeterangan

detections[].confidence number required 0–100 Persentase kepercayaan model AI (0 - 100)

GET /forum/posts/search Cari postingan berdasarkan judul atau tag optionalAuth ▼
Query Parameters
ParameterTipeValidasiKeterangan
titlestringoptionalFilter judul (substring match, case-insensitive)
tagsstringoptionalTag dipisah koma: acne,tips,skincare (case-insensitive, max 10 tag)
Contoh Request
URL
GET /forum/posts/search?title=jerawat&tags=acne,tips
Deskripsi Logika Filter
🔍
Cara kerja pencarian
Filter tags diproses di Firestore dengan array-contains-any (maks. 10 tag). Filter title diproses in-memory setelah data diambil. Hasil diurutkan berdasarkan net vote score (upvote - downvote) tertinggi.

  "status": "success",

}

👍 Forum — Votes

Sistem vote toggle otomatis. Satu user hanya bisa punya satu vote aktif per post.

POST /forum/posts/:postId/vote Berikan / toggle / hapus vote pada postingan requireAuth ▼
Logika Toggle Vote
➕ Belum ada vote
Vote baru ditambahkan. Counter upvoteCount atau downvoteCount diinkrementasi.
❌ Vote sama dikirim lagi
Vote dihapus (netral). Counter didekrementasi kembali ke posisi semula.
🔄 Vote berbeda dikirim
Vote diubah. Counter lama didekrementasi, counter baru diinkrementasi secara atomic.
Request Body (application/json)
FieldTipeValidasiNilai yang Valid
type string required "upvote" atau "downvote"
Response — 200 OK (semua skenario)
JSON · Vote Berhasil Ditambahkan
{
  "code": 200,
  "status": "success",
  "message": "Berhasil melakukan upvote"
}
JSON · Vote Dihapus (Toggle Off)
{
  "code": 200,
  "status": "success",
  "message": "Vote berhasil dihapus"
}
JSON · Vote Diubah
{
  "code": 200,
  "status": "success",
  "message": "Vote berhasil diubah menjadi downvote"
}
